🏗️ CUDA Stream 基础架构

什么是 CUDA Stream？

  • Stream 内的操作顺序执行
  • 不同 Stream 间的操作可以并行
  • 默认 Stream (Stream 0) 会阻塞其他 Stream
  • 使用 non_blocking=True 实现异步传输
  • Pinned Memory 是异步传输的前提条件

关键 API

# 创建非阻塞 Stream
stream = torch.cuda.Stream()

# 在指定 Stream 上执行
with torch.cuda.stream(stream):
    buf.copy_(data, non_blocking=True)
    result = compute(buf)

# Stream 同步
stream.synchronize()

# 跨 Stream 等待
stream_b.wait_stream(stream_a)

📋 Baseline：顺序执行模式

baseline_streams.py

def benchmark_fn(self):
    # 逐段处理：H2D → Sync → Compute → Sync
    for i in range(self.num_chunks):
        # ❶ 阻塞式 H2D 传输
        self.device_data[i].copy_(
            self.host_data[i]
        )
        # ❷ GPU 计算 (传输已完成)
        self.results[i] = self._compute(
            self.device_data[i]
        )
    # 每个 chunk 必须等待前一个完成
    # → 无重叠，大量 GPU 空闲

执行时序分析

  • 20 个 chunk × (H2D + Compute)
  • 每个 chunk 阻塞等待传输完成
  • GPU SM 在 H2D 期间 完全空闲
  • 内存控制器在计算期间 完全空闲
  • 总时间 ≈ Σ(H2Di) + Σ(Computei)
⚠️ 问题

5M 元素/chunk × 20 chunks，串行执行使 GPU 利用率极低

🚀 Optimized：双缓冲流水线

optimized_streams.py

def benchmark_fn(self):
    # ❶ 启动首次 H2D 传输
    with torch.cuda.stream(self.stream_h2d):
        self.device_data[0].copy_(
            self.host_data[0],
            non_blocking=True
        )

    for i in range(self.num_chunks):
        # ❷ 等待当前 chunk 传输完成
        self.stream_compute.wait_stream(
            self.stream_h2d
        )
        # ❸ 提前启动下一 chunk 传输
        if i + 1 < len(self.chunks):
            with torch.cuda.stream(self.stream_h2d):
                self.device_data[i+1].copy_(
                    self.host_data[i+1],
                    non_blocking=True
                )
        # ❹ 在 Compute Stream 上计算
        with torch.cuda.stream(self.stream_compute):
            self.results[i] = self._compute(
                self.device_data[i]
            )

🌀 Warp Specialization + Multi-Stream

概念：Warp 级别的流水线分工

  • Producer Warps：负责数据搬运 (TMA/UTCCP)
  • Consumer Warps：负责 Tensor Core 计算
  • 通过 DSMEM (Distributed SMEM) 在 Warp 间传递数据
  • Multi-Stream 放大：不同 Stream 上运行不同的 Warp 专业化 kernel
  • SM 级别的 overlap，而非仅 kernel 级别
DSMEM Producer-Consumer 1.67x 加速

执行模型

// Warp Specialized Kernel (CUDA)
__global__ void ws_kernel() {
  if (threadIdx.x < PRODUCER_THREADS) {
    // Producer: 数据搬运
    while (has_work) {
      tma_load(smem_buf, gmem_src);
      dsme_barrier_arrive();
    }
  } else {
    // Consumer: 计算
    while (has_work) {
      dsme_barrier_wait();
      tcgen05_mma(acc, smem_buf);
    }
  }
}
