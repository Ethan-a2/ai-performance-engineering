核心问题

低精度不仅仅是 dtype 选择 — 它是一个系统级决策

🎯 关键问题

"哪些运行时精度和内存管理变更，能在真实工作负载上带来足够大的收益，以证明所增加的复杂度是值得的？"

📐 设计原则

  • 某些收益来自 计算成本降低
  • 某些收益来自 内存带宽节省
  • 某些收益来自 重叠/流水线行为
  • 所有收益必须通过 同一 harness 验证
💡 低精度应被理解为一个 systems decision，而非单纯的 dtype 选择。部分收益来自计算减少，部分来自内存流量降低或更好的重叠行为。

BASELINE

手动 NVFP4 GEMM

  • 逐元素解包 FP4 → FP32
  • 标量累加 sum += a * b
  • 朴素 16×16 线程块
  • 无 Tensor Core 利用
OPTIMIZED

cuBLASLt NVFP4 Tensor Core

  • 原生 NVFP4 (E2M1) 硬件支持
  • Block-scaled FP8 量化因子
  • Tensor Core 矩阵乘法
  • cuBLASLt 自动调优

def _adaptive_cache_update(self, bits):
    if bits == 8:
        self._packed.copy_(self._int8_src)
    elif bits == 6:
        self._packed[..., :6].copy_(self._int6_src)
    elif bits == 4:
        self._packed[..., :4].copy_(self._int4_src)

INT4 / INT6 打包算法

INT4 打包 — 2 值/byte

def _pack_int4(values, out):
    """[-8, 7] → uint8, 每byte存2个值"""
    unsigned = (values & 0x0F).to(torch.uint8)
    pairs = unsigned.view(-1, 2)
    out.copy_((pairs[...,0] << 4) | pairs[...,1])

def _unpack_int4_to_float(packed, scale, out):
    high = packed >> 4          # 低4位
    low  = packed & 0x0F        # 高4位
    # 减去偏移 → 乘以 scale → 还原浮点

存储开销: 512 dim → 256 bytes + 4 bytes scale
压缩比: 8× vs FP32

INT6 打包 — 4 值→3 bytes

def _pack_int6(values, out):
    """[-32, 31] → 3 bytes per 4 values"""
    unsigned = (values + 32).to(torch.uint8)
    groups = unsigned.view(-1, 4)
    # 4×6bit = 24bit = 3×8bit
    out[...,0] = g[0] | ((g[1]&0x03)<<6)
    out[...,1] = ((g[1]>>2)&0x0F)|((g[2]&0x0F)<<4)
    out[...,2] = ((g[2]>>4)&0x03) | (g[3]<<2)

存储开销: 512 dim → 384 bytes + 4 bytes scale
压缩比: ~5.3× vs FP32

⚠️ 打包/解包使用 bit 操作在 GPU 上执行，确保量化因子 (scale) 保持 FP32 精度以避免精度漂移。

自适应并行策略

Tensor / Pipeline / Hybrid / Data 并行的动态选择

🔀 并行策略路由

class AdaptiveParallelismStrategy:
    def select(self, request):
        # 基于请求特征选择策略
        if request.seq_len > threshold:
            return "pipeline"   # 长序列
        elif request.batch > threshold:
            return "data"       # 大批量
        elif request.model_size > threshold:
            return "tensor"     # 大模型
        else:
            return "hybrid"     # 混合

🏊 Worker Pool 管理

class AdaptiveParallelismWorkerPool:
    def __init__(self, num_workers):
        self.pools = {
            "tensor": TensorPool(num_workers),
            "pipeline": PipelinePool(num_workers),
            "hybrid": HybridPool(num_workers),
            "data": DataPool(num_workers),
        }

    def dispatch(self, request):
        strategy = self.strategy.select(request)
        return self.pools[strategy].submit(request)
💡 在同一合成请求流上，baseline Python 路由循环与优化的向量化路由路径保持输出等价。ch19:adaptive_parallelism benchmark 验证这一等价性。

class MemoryAllocatorWithMonitoring:
    def allocate(self, size):
        ptr = self.backend.alloc(size)
        self.track(ptr, size)  # 追踪
        self.check_fragmentation()
        return ptr

    def check_fragmentation(self):
        frag = self.compute_fragmentation()
        if frag > threshold:
            self.defragment()  # 碎片整理
