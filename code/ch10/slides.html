Architecture

Blackwell sm_100 核心特性

🔧 TMA (Tensor Memory Accelerator)

硬件级异步内存拷贝引擎，支持 2D/3D 张量描述符，零 SM 开销的 Global → Shared 数据搬运。

// TMA 描述符 + 异步拷贝
cudaTensorMapEncodeTiled(...)
cp_async_bulk_tensor(smem, tma_desc, ...)

🧩 tcgen05 Tensor Core

第五代 Tensor Core 指令集，支持 FP16/BF16/FP8/NVFP4，Warp Group 级 MMA 操作。

// tcgen05 MMA 指令
tcgen05::mma(d, a, b, ...)
tcgen05::commit(...)

🏗️ Thread Block Clusters

多 CTA 协作执行，DSMEM (Distributed Shared Memory) 跨 CTA 通信，TMA Multicast 广播。

// Cluster 级操作
cluster.sync()
tma_multicast(smem, desc, ...)

⚡ Warp Specialization

Producer Warp 专司数据加载，Consumer Warp 专司 Tensor Core 计算，消除角色切换开销。

// Warp 角色分工
if (warp_id < producer_count)
  producer_loop();
else
  consumer_loop();

🔄 Pipelines

多级异步流水线：双缓冲、三阶段、TMA Micro-Pipeline，利用 async barrier 实现计算与加载重叠。

// 多阶段流水线
stage0: load(next_tile)
stage1: compute(cur_tile)
async_barrier()

🔁 Persistent Kernels

内核内循环，一次启动处理所有工作块，消除反复的 launch overhead。

// 持久化内核循环
while (work_tile = get_next_tile()) {
  process_tile(work_tile);
}

Pipeline Deep Dive

异步流水线架构演进

Stage 1: Double Buffered Pipeline

// baseline: 同步加载 + 计算
for each tile:
  load(tile)          // SM 等待
  compute(tile)       // 内存空闲

// optimized: 双缓冲重叠
load(buf[0])          // 预加载第一块
for each tile:
  load(buf[next])     // 异步加载下一块
  compute(buf[cur])   // 同时计算当前块
  swap(buf[cur], buf[next])

Stage 3: 3-Stage Pipeline

// 三级流水: Global → Register → Compute
stage0: cp.async(gmem → smem[buf0])
stage1: ldmatrix(smem → reg[buf1])
stage2: mma.sync(reg[buf2])
async_barrier()  // 流水线同步

Stage 2: TMA 2D Pipeline

// TMA 硬件加速 2D 张量拷贝
cudaTensorMapEncodeTiled(
  &desc, float,
  rank=2,
  globalDim=[M, N],
  boxDim=[TILE_M, TILE_N]
)

// 内核中零 SM 开销搬运
cp_async_bulk_tensor(
  smem_ptr, desc, coord
)

Stage 4: TMA Micro-Pipeline (FlashAttention)

// FlashAttention TMA 微流水线
// Q/K/V 三路 TMA 异步加载
tma_load(Q_tile, stage=0)
tma_load(K_tile, stage=1)
tma_load(V_tile, stage=2)
// 计算与加载完全重叠
flash_attention_compute()

Warp Specialization

Warp 角色特化：从混合到分工

❌ Baseline: 混合角色

Warp 0: Load → Compute → Store
Warp 1: Load → Compute → Store
Warp 2: Load → Compute → Store
Warp 3: Load → Compute → Store

每个 Warp 都在角色切换，Tensor Core 利用率低

✅ Optimized: 角色特化

Warp 0-1: Producer (TMA Load)
Warp 2-3: Consumer (Tensor Core MMA)

专用 Producer 持续喂数据，Consumer 持续计算

Ping-Pong Staging

// 乒乓缓冲 + Warp 特化
__shared__ float buf[2][TILE];

// Producer Warp
for (stage = 0; stage < num_stages; stage++) {
  cp.async(buf[stage % 2], gmem);
  arrive(load_barrier);
  wait(compute_barrier);
}

// Consumer Warp
for (stage = 0; stage < num_stages; stage++) {
  wait(load_barrier);
  mma(reg, buf[stage % 2]);
  arrive(compute_barrier);
}

Persistent Kernels

持久化内核：一次启动，全部完成

❌ Non-Persistent (Baseline)

// 每个 tile 一次 kernel launch
for tile in tiles:
  kernel<<>>(tile)
  // launch overhead: ~5-10 μs
  // 100 tiles = 500-1000 μs 开销!

Launch overhead 在小 kernel 或多 tile 时成为瓶颈

✅ Persistent (Optimized)

// 一次 launch，内核内循环
__global__ void persistent_kernel() {
  while (true) {
    int tile = atomicAdd(&counter, 1);
    if (tile >= num_tiles) break;

    process_tile(tile);
  }
}
// launch overhead: 仅 1 次!

Cooperative Persistent + TMA Pipeline

// 持久化 + 两阶段 SM 流水线
__global__ void coop_persistent() {
  __shared__ float smem[2][TILE];

  while (has_work) {
    // Stage 0: TMA 异步加载
    tma_load_async(smem[buf], tile);
    cp_async_barrier();

    // Stage 1: 计算
    compute(smem[1-buf]);
    __syncthreads();
  }
}

Persistent MatMul with TMA

// 持久化矩阵乘 + TMA
// 设备端 tile 调度
while ((tile_m = fetch_tile()) < M_tiles) {
  for (tile_n = 0; tile_n < N_tiles; tile_n++) {
    tma_load(A_tile[tile_m][k]);
    tma_load(B_tile[k][tile_n]);
    mma(C_acc, A_smem, B_smem);
  }
}

FlashAttention3 Pipeline Code

// Warp 特化 FlashAttention
// Producer: 加载 Q/K/V
if (warp_id < 2) {
  while (tile < num_tiles) {
    tma_load(K_tile[tile]);
    tma_load(V_tile[tile]);
    arrive(load_bar);
    wait(compute_bar);
    tile++;
  }
}
// Consumer: 计算 Attention
else {
  while (tile < num_tiles) {
    wait(load_bar);
    S = Q @ K^T / sqrt(d);
    P = softmax(S);
    O += P @ V;
    arrive(compute_bar);
    tile++;
  }
}

Cluster Multicast

TMA Multicast GEMM

Baseline: 逐 CTA 加载

CTA 0: Load A tile
CTA 1: Load A tile
CTA 2: Load A tile
CTA 3: Load A tile

同一 A tile 被加载 4 次！带宽浪费

Optimized: TMA Multicast

TMA Multicast A tile → CTA 0,1,2,3

一次 TMA 操作广播到所有 CTA，NVLink-C2C 加速

Multicast GEMM 核心代码

// TMA Multicast GEMM
uint32_t cta_mask = 0xFFFF; // 所有 CTA

// A tile multicast (广播)
tma_multicast(
  smem_A, desc_A, coord_A,
  cta_mask
);
// 所有 CTA 同时收到 A tile

// B tile 各自加载 (不共享)
tma_load(smem_B, desc_B, coord_B);

cluster.sync();

// 所有 CTA 独立计算
tcgen05::mma(C, smem_A, smem_B);

适用场景

  • GEMM: A 矩阵被多 CTA 共享行/列
  • Attention: Q 被多 head 共享
  • Convolution: 权重被多 output tile 共享
  • 任何 读多写少 的共享数据模式
