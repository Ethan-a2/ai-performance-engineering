🎯 The Problem

Optimization decisions must be grounded in measured hardware behavior, not marketing specs.

  • Untuned cuBLAS defaults

This chapter is the hardware sanity anchor for later claims.

💾 Memory Transfer: PCIe vs Pinned

Baseline: Pageable Memory

self.host_data = torch.randn(
  self.N, dtype=torch.float32,
  pin_memory=False
)

# Synchronous copy
self.device_data.copy_(
  self.host_data, non_blocking=False
)

Result: 18.901 ms

Optimized: Pinned Memory

self.host_data = torch.randn(
  self.N, dtype=torch.float32,
  pin_memory=True
)

# Asynchronous copy
self.device_data.copy_(
  self.host_data, non_blocking=True
)

Result: 3.637 ms

5.20x Speedup

Why? Pinned memory enables DMA transfers without CPU staging, and async copy overlaps computation.

🔢 cuBLAS: FP32 vs TF32

Baseline: Pure FP32

# Disable TF32
torch.backends.cuda
  .matmul.allow_tf32 = False

# Standard FP32 matmul
C = torch.matmul(A, B)

Result: 0.590 ms

Precision: FP32 only

Optimized: TF32 Enabled

# Enable TF32 via backend policy
backend_policy="performance"

# cuBLAS routes through
# tensor cores (TF32)
C = torch.matmul(A, B)

Result: 0.114 ms

5.17x Speedup

Precision: TF32 (tensor cores)

Key Insight: TF32 uses tensor cores with 10-bit mantissa, providing FP32 range with sufficient precision for most ML workloads.

🔗 Grace-Blackwell Coherent Memory

🚀

23.14x Speedup on GB200/GB300

🎯 Zero-Copy Buffers

Sub-MB transfers benefit from shared memory mappings without explicit copies.

⚡ NVLink-C2C

Direct coherent link between Grace CPU and Blackwell GPU eliminates PCIe bottlenecks.

🔄 Cache Coherency

Hardware-managed coherency ensures consistent views across CPU and GPU caches.

⚠️ Platform Requirement

Only available on Grace-Blackwell systems. PCIe-only hosts fail fast with SKIPPED diagnostic.

Important: On non-Grace hosts, the benchmark stops immediately with an explicit unsupported-capability diagnostic instead of emitting fallback numbers.
