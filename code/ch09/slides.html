Attention Fusion: SDPA

3 kernels → 1 fused FlashAttention-style kernel

Baseline: 3 Kernels

# Kernel 1: Q @ K^T → HBM attn_scores = matmul(Q, K.transpose()) # Scale attn_scores = attn_scores * scale # Kernel 2: softmax → HBM attn_weights = softmax(attn_scores) # Kernel 3: attn @ V → HBM output = matmul(attn_weights, V)

2 HBM round-trips for intermediate tensors (S×S matrix)

Optimized: Fused

# Single fused kernel # Q, K, V loaded once from HBM # Tiling: compute Q@K^T in SRAM # Online softmax (no full materialization) # Accumulate attn@V incrementally # Single write-back to HBM output = flash_attention(Q, K, V)

S×S attention matrix never leaves SRAM — 1.71× speedup

Micro-Tiling & Register Blocking

Balancing register pressure with data reuse for tensor cores

Key Techniques

  • Explicit register blocking (TM × TN tiles)
  • cp.async prefetch from global → shared
  • Double-buffered shared memory
  • Warp-level MMA with wmma / mma.sync
  • Occupancy tuning (registers vs threads)

tcgen05 Pipeline

  • Blackwell tensor-core generation 05
  • TMA (Tensor Memory Accelerator) loads
  • Producer/consumer async pipeline
  • Staged loads with inline PTX hooks
  • Architecture-gated intrinsics

Triton: Rapid Prototyping

Python-level kernel authoring with compiler-generated PTX on Blackwell

Baseline (PyTorch)

# Standard PyTorch elementwise def baseline_elementwise(x, out): out.copy_(x * 2.0 + 1.0)

Launches 2 kernels (mul + add), reads/writes HBM twice

Optimized (Triton)

# Single Triton kernel @triton.jit def fused_elementwise(X, OUT, N): pid = tl.program_id(0) offs = pid * BLOCK + tl.arange(0, BLOCK) x = tl.load(X + offs) tl.store(OUT + offs, x * 2.0 + 1.0)

1 kernel, 1 HBM read + write, vectorized loads

Learning Goals

1. Roofline Reasoning

Separate compute-bound vs memory-bound behaviors. Adjust kernels to move workloads toward the roofline knee.

2. Micro-Tiling Design

Design schedules that balance register pressure with data reuse. Use cp.async prefetch and double-buffered shared memory.

3. CUTLASS & Triton

Leverage template metaprogramming (CUTLASS) and Python-level kernel authoring (Triton) for rapid iteration with custom CUDA fallbacks.

4. Kernel Fusion

Fuse reduction-heavy kernels (norm + activation) to eliminate redundant memory trips. Use online algorithms to avoid materializing intermediates.
