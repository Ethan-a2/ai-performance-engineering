基准测试框架

生产级基准测试执行系统

核心特性

  • 自动发现264+基准测试对
  • 子进程隔离执行
  • 超时保护和错误恢复
  • CUDA事件精确计时
  • 统计分析（均值、中位数、百分位数）

  • 线程模式：快速原型开发
  • Torchrun模式：多GPU分布式

基准测试方法论

三层基准测试方法

1. 微基准测试

隔离子系统限制和回归

  • 计算、通信、存储分析
  • 内核级性能分析
  • 硬件限制识别

2. 组件基准测试

测量服务、数据加载器、调度器路径

  • 固定工作负载语义
  • 子系统回归识别
  • 用户可见路径分析

3. 端到端基准测试

在理解下层后验证现实工作流

  • 实际用户体验测量
  • 完整系统交互
  • 队列和调度影响
工作负载冻结规则：在比较任何内容之前，冻结工作负载定义： - 相同模型 - 相同序列长度组合 - 相同精度 - 相同批处理策略 - 相同数据集或提示语料库

MCP工具

AI助手集成工具

核心工具

  • gpu_info：获取GPU硬件信息
  • analyze_bottlenecks：识别性能问题
  • optimize：运行快速LLM基准测试变体
  • benchmark_explore：复制基线基准测试，运行变体
  • recommend：获取优化建议
  • ask：询问性能问题

工作流程

# 深度比较基线 vs 优化 { "targets": ["ch10:atomic_reduction"], "output_dir": "artifacts/runs", "async": true } # 标准基准测试运行 { "targets": ["ch10:atomic_reduction"] }

安全说明：MCP服务器通过stdio使用JSON-RPC，用于本地使用（编辑器集成、本地自动化、CI）。

验证系统

95个有效性问题保护

关键保护

  • 计时保护（流同步、事件计时）
  • 输出验证（常量输出、陈旧缓存）
  • 工作负载完整性（精度不匹配、隐藏降采样）
  • 位置验证（CPU溢出、设置预计算）
  • 内存跟踪（预分配输出、输入输出别名）
  • CUDA保护（异步Memcpy、工作区预计算）
