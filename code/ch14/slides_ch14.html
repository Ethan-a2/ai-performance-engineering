❌ 传统方式的痛点

  • Graph breaks 破坏优化连续性

  • 减少内核启动次数
  • 融合计算图 (graph fusion)
  • 优化内存访问模式
  • 持续化内核调度

⚡ torch.compile 深度解析

编译流程

# Eager baseline model = SimpleTransformer() output = model(input_ids) # torch.compile optimized model = torch.compile( SimpleTransformer(), mode="reduce-overhead" ) output = model(input_ids)

编译模式

  • default: 平衡编译时间和运行时性能
  • reduce-overhead: 最大化运行时性能
  • max-autotune: 最激进的优化

优化原理

Phase 1: 图捕获

通过 tracing 或 scripting 捕获计算图

Phase 2: 图优化

融合算子、消除冗余、优化内存

Phase 3: 代码生成

生成优化的 Triton/CUDA 内核

关键指标：
减少 graph breaks → 更大的优化区域

🔄 Triton 持久化内核

Standard vs Persistent

Standard GEMM

# 一个 block 处理一个 tile grid = (M//TILE_M, N//TILE_N) # 启动数千个 blocks

启动开销大，L2 复用差

Persistent GEMM

# 一个 SM 处理多个 tiles grid = (NUM_SMS, 1) # 每个 block 循环处理 tiles

减少启动，提升 L2 复用

核心优势

  • 减少内核启动开销 (1 vs 数千次)
  • 跨 tile 的 L2 缓存复用
  • 工作窃取实现负载均衡
  • 小尺寸矩阵收益显著

实现模式

@triton.jit def gemm_persistent_kernel( a_ptr, b_ptr, c_ptr, M, N, K, # ... NUM_SMS: tl.constexpr, ): # 持久化循环 pid = tl.program_id(0) num_tiles = num_tiles_m * num_tiles_n for tile_id in range( pid, num_tiles, NUM_SMS ): # 处理当前 tile # Swizzle 模式优化 L2 复用

适用场景

小尺寸 GEMM
启动开销主导
需要动态负载均衡
StreamK 算法

🧠 Attention 优化策略

Baseline: Per-Head Serial

# 逐头串行计算 for head in range(num_heads): qh = q[:, head, :] kh = k[:, head, :] vh = v[:, head, :] # 每个 head 独立计算 scores = matmul(qh, kh.T) * scale attn = softmax(scores, dim=-1) output = matmul(attn, vh)
16 次独立内核调用

Optimized: Fused SDPA

# 使用 PyTorch SDPA import torch.nn.functional as F # 单次融合调用 output = F.scaled_dot_product_attention( q, k, v, attn_mask=None, is_causal=True )
1 次融合调用

Flash Attention

分块计算，减少 HBM 访问

内存效率

O(N) 内存 vs O(N²) 传统

精度保持

数值稳定的 softmax 实现

📍 区域编译 (Regional Compilation)

概念

不是编译整个模型，而是编译计算图中稳定的子区域

优势

  • 减少编译时间
  • 更细粒度的优化
  • 处理动态控制流
  • 增量编译支持

适用场景

图中有动态 shapes
部分图不稳定
需要快速编译

实现示例

import torch # 定义稳定的子模块 class StableBlock(nn.Module): def forward(self, x): return self.ffn(self.attn(x)) # 区域编译 @torch.compile def compiled_block(x): return stable_block(x) # 混合执行 for layer in model.layers: if layer.is_stable: x = compiled_block(x) else: x = layer(x) # Eager

📡 NCCL 量化通信

问题

多 GPU 通信是分布式训练的主要瓶颈

解决方案

通信量化

在通信前压缩数据 (FP32 → FP16/INT8)

流水线重叠

计算与通信重叠执行

DeepSeek L2 Bypass 创新

L2 Cache Bypass ┌─────────────┐ │ SM 计算 │ └──────┬──────┘ │ 直接写入 HBM ▼ ┌─────────────┐ │ HBM │ └─────────────┘ 跳过 L2，减少缓存污染

适用于：大模型通信密集场景

🚀 TMA: Tensor Memory Accelerator

什么是 TMA？

Blackwell 架构的硬件内存搬运引擎

核心能力

  • 异步内存搬运
  • 多维张量模式支持
  • 硬件加速的地址计算
  • 与计算重叠执行

Triton 集成

# TMA 描述符创建 desc = tl.make_tensor_desc( ptr, shape, strides, block_shape ) # 异步加载 tile = desc.load(offsets)

性能影响

2-3x
内存带宽利用率提升

配置参数

参数 说明

实现示例

from torch.nn.attention.flex_attention import ( flex_attention, create_block_mask ) # 定义自定义 score mod def score_mod(score, b, h, q_idx, kv_idx): # 滑动窗口 return torch.where( (q_idx - kv_idx).abs() < WINDOW_SIZE, score, float('-inf') ) # 使用 FlexAttention output = flex_attention( q, k, v, score_mod=score_mod )
