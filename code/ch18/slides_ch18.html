FlexDecoding: Work Reduction Story

Baseline Problem

Scores the full KV cache with a sliding-window mask.

Q @ K^T → full [seq_len × seq_len]
Apply sliding-window mask → O(seq²) compute
Most work discarded by mask

Optimized Solution

Trims decode step to active window before attention.

Slice K/V to window [pos-w : pos]
Q @ K_sliced^T → O(w) compute
No wasted masked-out work
1.97× SPEEDUP
161.6 → 82.0 ms LATENCY

Tensor Core Acceleration

Baseline: FP32 MatMul

# FP32 → no tensor cores
A = torch.randn(4096, 4096,
    dtype=torch.float32)
B = torch.randn(4096, 4096,
    dtype=torch.float32)
out = A @ B  # CUDA cores

Standard FP32 FMA units — no hardware acceleration for matrix ops.

Optimized: FP16 Tensor Cores

# FP16 → tensor cores
A = torch.randn(4096, 4096,
    dtype=torch.float16)
B = torch.randn(4096, 4096,
    dtype=torch.float16)
out = A @ B  # Tensor Cores!

Tensor cores: 4×4×4 FMA per clock — massive throughput for matmul.

15.65× SPEEDUP
3.805 → 0.243 ms LATENCY

FlashMLA Decode Kernel (Blackwell sm_120)

__global__ void flashmla_decode(
    const half* q,          // [B, H, D]
    const half* k_cache,    // [B, S, H, D]
    const half* v_cache,    // [B, S, H, D]
    half*       out,        // [B, H, D]
    const int*  lengths,    // [B] seq lengths
    int num_heads, int head_dim, int stride)
{
  // Per-head attention: Q·K scores → softmax → V weighted sum
  half acc = 0, denom = 0;
  for (int pos = 0; pos < lengths[batch]; ++pos) {
    score = __hfma(q[d], k[pos][d], score);  // FMA
    weight = hexp(score / sqrt(head_dim));    // softmax
    acc = __hfma(weight, v[pos][d], acc);     // accumulate
  }
  out = acc / denom;  // normalize
}

Illustrative sketch — production uses FlashMLA/ThunderMLA or PyTorch FlexDecoding kernels. Includes Blackwell TMEM guard for SM121 compilation.

Key Takeaways

Work Reduction (moderate)

FlexDecoding: slice KV to active window.

~2× FLEXDECODING

Kernel-Level (big wins)

Tensor cores + fused RoPE/Q/cache.

~16× TENSOR CORES
~24× ROPE CACHE

Serving Integration

vLLM decode graphs eliminate per-step launch overhead. Paged attention manages KV memory at scale.

Lesson

Don't average "moderate" and "big kernel" improvements together — they're different stories. Measure on your hardware before treating numbers as decision thresholds.
