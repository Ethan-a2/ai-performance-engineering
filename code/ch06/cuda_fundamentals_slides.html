CUDA Kernel Basics

Simple Kernel (my_first_kernel.cu)


__global__ void double_values(float* data, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        data[idx] *= 2.0f;
    }
}

Key Concepts

  • __global__: Device function callable from host
  • blockIdx: Block index in grid
  • threadIdx: Thread index in block
  • Grid/Block hierarchy for parallelism

Sequential vs Parallel Execution

Baseline: Sequential (baseline_add_cuda.cu)


__global__ void addSequential(
    const float* A, const float* B, 
    float* C, int n) {
    if (blockIdx.x == 0 && threadIdx.x == 0) {
        for (int i = 0; i < n; ++i) {
            C[i] = A[i] + B[i];
        }
    }
}

❌ Only 1 thread does all work

Optimized: Parallel (optimized_add_cuda_parallel.cu)


__global__ void addParallel(
    const float* A, const float* B, 
    float* C, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        C[idx] = A[idx] + B[idx];
    }
}

✅ All threads work in parallel

3881x speedup from proper parallelization

Instruction-Level Parallelism (ILP)

Baseline: 1 chain per thread


// baseline_attention_ilp.py
self._extension.sequential_ops(dst, src)
// Low ILP: dependent operations

❌ Limited instruction parallelism

Optimized: 4 chains per thread


// optimized_attention_ilp.py
self._extension.unrolled_ilp(dst, src)
// High ILP: 4 independent chains

✅ 4-way ILP for better throughput

265.82x speedup from increased ILP

Shared Memory Bank Conflicts

Baseline: Bank Conflicts


// baseline_bank_conflicts.cu
__shared__ float shared_data[TILE_VALUES];
// All threads access same bank
acc += shared_data[logical_offset];

❌ Warp serialization due to bank conflicts

Optimized: Padded Memory


// optimized_bank_conflicts.cu
__shared__ float shared_data[PADDED_VALUES];
// Padded stride avoids conflicts
int padded_idx = col * PADDED_STRIDE + row;
acc += shared_data[padded_idx];

✅ Unique bank per thread lane

Key: PADDED_STRIDE = NUM_BANKS + 1

Memory Optimization Patterns

  • Vectorized Access: Use float4 for coalesced memory access
  • Shared Memory: Reduce global memory bandwidth
  • Bank Conflict Avoidance: Pad shared memory arrays
  • Memory Coalescing: Adjacent threads access adjacent addresses
  • Unified Memory: Automatic data migration between CPU/GPU

Key insight: "Memory layout matters as much as computation"
