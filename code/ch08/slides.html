Threshold: Code Comparison

Baseline: Branch-Heavy

// 4-way branch → warp divergence if (value > outer_threshold) { // Path A: outer positive result = compute(value, 1.25f); } else if (value > threshold) { // Path B: inner positive result = compute(value, 0.85f); } else if (value < -outer) { // Path C: outer negative result = compute(value, -1.25f); } else if (value < -threshold) { // Path D: inner negative result = compute(value, -0.85f); } // Threads idle during other paths!

Optimized: Predicated

// No branches → full warp utilization float scale = compute_threshold_scale( value, threshold); // Predicated: all threads execute if (scale == 0.0f) { return 0.0f; // uniform path } float sine, cosine; __sincosf(value, &sine, &cosine); return expensive_transform( value, sine, cosine) * scale; // All threads execute same path!

关键差异：Baseline 使用 4-way nested if/else 导致 warp divergence；Optimized 使用 predication 让所有线程执行相同指令序列

Loop Unrolling: ILP Optimization

通过 ILP 和向量化加载减少 execution-dependency stalls

Baseline

// 标量加载，冗余累加 for (k = 0; k < 512; ++k) { float mul = row[k] * weights[k]; // 16 次冗余累加 for (r = 0; r < 16; ++r) { sum += mul * (1.0f/16.0f); } } // 问题：依赖链长，ILP 低
1.591 ms 低 ILP 标量加载

Optimized

// 向量化加载 + 4 个独立累加器 float4 vec = load_float4(row + idx); float acc0, acc1, acc2, acc3; // 4 路独立 FMA acc0 = fmaf(vec.x, w.x, acc0); acc1 = fmaf(vec.y, w.y, acc1); acc2 = fmaf(vec.z, w.z, acc2); acc3 = fmaf(vec.w, w.w, acc3); // Shared memory 缓存权重 // Warp shuffle 归约
0.382 ms 4.17× 快 高 ILP

Loop Unrolling: Key Techniques

1. Vectorized Loads

float4* row_vec = reinterpret_cast<float4*>( row_ptr); // 1 load = 4 floats

将 4 次标量加载合并为 1 次向量加载，减少内存事务

2. Multiple Accumulators

float acc0 = 0; float acc1 = 0; float acc2 = 0; float acc3 = 0; // 4 independent chains

打破依赖链，允许 GPU 同时执行多条独立指令

3. Shared Memory Cache

extern __shared__ float shared_weights[]; // Warp shuffle broadcast weight[w] = __shfl_sync( mask, lane_weight, w);

权重缓存到 shared memory，减少全局内存访问

ILP 原理

GPU 每周期可以发射多条独立指令。当一个累加器等待 FMA 结果时，scheduler 可以切换到另一个累加器的指令，从而隐藏延迟。

AI Optimization: Occupancy + ILP

AI kernel 调度示例：shared weights + 向量化 + occupancy 优化

Baseline Kernel

__global__ void ai_baseline( float* inputs, float* weights, float* output, int rows, int cols) { int row = blockIdx.x * blockDim.x + threadIdx.x; float acc = 0.0f; // 逐元素加载权重（全局内存） for (int col = 0; col < cols; ++col) { acc = fmaf(row_ptr[col], weights[col], acc); } output[row] = tanhf(acc); }

Optimized Kernel

__global__ void ai_optimized( float* inputs, float* weights, float* output, int rows, int cols) { // Step 1: 权重加载到 shared memory extern __shared__ float sw[]; for (int i = threadIdx.x; i < cols; i += blockDim.x) sw[i] = weights[i]; __syncthreads(); // Step 2: 4-way 向量化累加 float4* row_vec = ...; float acc0, acc1, acc2, acc3; // Step 3: 独立累加器链 #pragma unroll 4 for (int v = 0; v < vec_cols; ++v){ acc0 = fmaf(vals.x, w.x, acc0); acc1 = fmaf(vals.y, w.y, acc1); // ... } }

Occupancy Tuning Deep Dive

通过调整 CTA shape、寄存器上限和 launch 配置优化 occupancy

Tunable Parameters

// occupancy_tuning.cu int block_size = 128; // CTA 大小 int smem_bytes = 0; // Shared memory int unroll = 1; // 循环展开因子 int inner_iters = 2; // 内部迭代次数 // __launch_bounds__ 控制寄存器 __global__ __launch_bounds__(256, 4) void kernel(...) { // 4 个独立累加器 → ILP float a = val * 1.1f; float b = val * 0.9f; float c = val * 1.3f; float d = val * 0.7f; }

Occupancy vs ILP Trade-off

高 Occupancy：

  • 更多 CTA resident → 更好的 latency hiding
  • 但每个 CTA 资源更少（寄存器/shared memory）

高 ILP：

  • 更多独立指令 → 更好的 instruction throughput
  • 但需要更多寄存器 → 可能降低 occupancy

平衡点：通过 profiler 找到最佳的 occupancy/ILP 组合
