Deep Dive: Vectorization

1M elements, chunk size 4096 — Python loop vs. single GPU kernel.

Baseline Scalar Chunking

result = torch.zeros(1, device=device)
chunk = 4096
for start in range(0, N, chunk):  # 244 iterations!
    result += data[start:start+chunk].sum()
Problem: 244 Python loop iterations, each launching a tiny kernel. Python overhead dominates.

Optimized Vectorized

# Single kernel call — no Python loop
result = data.sum().unsqueeze(0)
Solution: One fused reduction kernel. Eliminates 243 kernel launches and all Python overhead.

Deep Dive: Storage IO Path

64MB tensor — pageable memory vs. mmap + pinned buffers.

Baseline Pageable CPU

# Every iteration: full load + new allocation
cpu_loaded = np.load(filepath)
data = torch.from_numpy(cpu_loaded).to(device)
Disk → np.load (alloc) → torch.from_numpy → CUDA memcpy (pinned?)

Optimized mmap + Pinned

# Setup: one-time buffer creation
host_buf = torch.empty(size, pin_memory=True)
device_buf = torch.empty(size, device=device)

# Benchmark: zero-alloc hot path
mapped = np.load(filepath, mmap_mode="r")
np.copyto(host_view, mapped)
device_buf.copy_(host_buf, non_blocking=True)
Disk → mmap view (O(1)) → pinned buf → async DMA
Optimization Effect
mmap_mode="r" Zero-copy file view, OS handles page faults
pin_memory=True Enables async DMA transfer to GPU
non_blocking=True Overlaps copy with compute on default stream
Reusable buffers Eliminates per-iteration allocation overhead

Deep Dive: AI Pipeline Overlap

256 inference blocks — sync reads vs. double-buffered prefetch.

Baseline: Sequential (Sync)

[Load block 0] → [Compute 0] → [Load block 1] → [Compute 1] → [Load block 2] → ... │◄── GPU idle ──►│ │◄── GPU idle ──►│ │◄── GPU idle ──►│

Optimized: Double-Buffer Overlap

[Prefetch 0] → [Prefetch 1 + Compute 0] → [Prefetch 2 + Compute 1] → ... │◄── overlapped ──►│ │◄── overlapped ──►│

Key Technique: Double Buffering

host_buffers = [buf0, buf1]   # pinned
device_buffers = [dev0, dev1]
copy_stream = torch.cuda.Stream()

# Pipeline: prefetch next while computing current
stage_to_host(next_slot, step+1)
enqueue_copy(next_slot)       # on copy_stream
out = block(current_input)    # on default stream

Stream Synchronization

# Default stream waits for copy_stream
current_stream.wait_stream(copy_stream)

# Non-blocking copy runs on dedicated stream
with torch.cuda.stream(copy_stream):
    device_buf.copy_(host_buf, non_blocking=True)

GPUDirect Storage (GDS)

Bypass CPU entirely: NVMe → GPU via DMA over PCIe.

NVMe SSD
→
PCIe DMA
Zero CPU copy
→
GPU VRAM

Traditional Path

NVMe → kernel buffer → user buffer → pinned buffer → GPU CPU copies: 3 CPU↔GPU copy: 1

4 memory copies, CPU bottleneck

GDS Path

NVMe → GPU VRAM (RDMA) Zero CPU involvement

1 DMA transfer, no CPU staging

Fail-Fast Design: gds_cufile_minimal.py either confirms usable cuFile/GDS support or exits with SKIPPED: — no host-mediated fallback numbers published under GDS name.

Key Takeaways

Do This

  • Vectorize preprocessing — eliminate Python loops
  • Use mmap_mode="r" for storage reads
  • Pin host buffers for async DMA transfers
  • Double-buffer to overlap IO with compute
  • Tune DataLoader workers & prefetch factor
  • Use GPU streams for pipeline parallelism
  • Evaluate GDS when NVMe is the bottleneck

Avoid This

  • Python loops over tensor chunks
  • Per-iteration memory allocation
  • Pageable memory for GPU transfers
  • Synchronous storage reads in hot paths
  • Publishing fallback numbers as GDS results
  • Ignoring GPU idle time in profiling
"GPUs do not care how elegant a kernel is if the input path is late."
